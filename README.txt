*******************  Περιγραφή δομών  *******************

- PoolNode.hpp : Το header που περιέχει τις δομές που κρατάνε
σημαντικές πληροφορίες για την εξέλιξη του προγράμματος.

PoolNode : Δομή που κρατάει τις μεταβλητές για τα pools, όπως 
τις δουλειές που τους έχουν ανατεθεί, τις ενεργές από αυτές, 
και το pid του κάθε pool.

Info : Δομή που κρατάει το συνολικό αριθμό δουλειών και pool 
που έχουν δημιουργηθεί. 

JobNode : Δομή που κρατάει τις μεταβλητές για τα jobs, όπως 
το λογικό id, και το pid της καθεμιάς, την κατάσταση, και το
χρόνο στον οποίο είναι ενεργή.

*******************  Περιγραφή προγραμμάτων (κυρίως)  *******************


- Console.cpp : Το πρόγραμμα για την υποβολή εργασιών και ερωτημάτων
  για αυτές : 

  	    @  Αντιστοίχιση ορισμάτων σε μεταβλητές.

  	    @  Άνοιγμα των named pipes για γράψιμο και διάβασμα.

  	    @  Διάβασμα από το operations file, και γράψιμο στο named pipe μέσα σε loop.

  	    @  Διάβασμα από το αντίστοιχο named pipe και προβολή στο stdout.

  	    @  Μόλις τελειώσει με το operations file : 

  	       @  Διάβασμα από το stdin, και γράψιμο στο named pipe μέσα σε loop.

  	       @  Διάβασμα από το αντίστοιχο named pipe και προβολή στο stdout.

  	    @  Κλείσιμο των named pipes.


- Coord.cpp : Το πρόγραμμα που διαχειρίζεται τις μεθόδους που 
  έχουν να κάνουν με τις εργασίες και τα ερωτήματα. Σύντομη περιγραφή μεθόδων : 

        @  Αντιστοίχιση ορισμάτων σε μεταβλητές.

  		  @  Δημιουργία και άνοιγμα των named pipes για γράψιμο και διάβασμα.

  		  @  Submit : Λειτουργεί αναλόγως του αριθμού των pools που έχουν δημιουργηθεί, 
           και των εργασιών που έχουν κατατεθεί στην καθεμία. Το Coord γίνεται suspend
           μέχρι να δημιουργήσει το pool τα κατάλληλα named pipes, και στη συνέχεια αφού
           γίνει η ανταλλαγή πληροφοριών μεταξύ των δύο, το pool θα μπει σε έναν κύκλο, 
           και θα δημιουργήσει το job, στο οποίο θα γίνουν όλες οι διαδικασίες για την 
           δημιουργία του αντίστοιχου directory, με τα αποτελέσματα της εκτέλεσης. Οι 
           μεταβλητές ανανεώνονται σε κάθε κύκλο, και εξυπηρετούν αποτελεσματικά τις 
           εντολές που διαβάζονται από το Console στο Coord.

  		  @  Status : Με ένα kilL(job_pid, 0) σήμα αποφασίζεται η κατάσταση του job
           που διαβάζουμε. Το job_pid διατηρείται στη δομή που αναφέρεται παραπάνω.

  		  @  Status-All : Αναλόγως των μεταβλητών που έχουν τροποποιηθεί κατάλληλα 
           κατά το submit, αποφασίζεται η κατάστασή τους.

  		  @  Show-Active : Παρομοίως με το παραπάνω. 
      
        @  Show-Pools : Εξαγωγή πληροφοριών από τις δομές, και ανάδειξη πλήθους 
           ενεργών εργασιών για κάθε pool. 

        @  Show-Finished : Παρομοίως με τα Status-All, Show-Active. 

        @  Suspend : Στέλνεται ένα SIGSTOP σήμα για την αναστολή της 
           λειτουργίας της εργασίας.

        @  Resume : Στέλνεται ένα SIGCONT σήμα για τη συνέχιση της 
           λειτουργίας της εργασίας.
      
        @  Shutdown : Στέλνεται SIGTERM σήμα σε κάθε pool, και σε κάθε
           ενεργή εργασία αν αυτή υπάρχει, όπως και γίνονται οι 
           κατάλληλες αποδεσμεύεις των names pipes.


******************* Διευκρινήσεις - Παρατηρήσεις  *******************


- Έχουν υλοποιηθεί όλες οι ζητούμενες λειτουργίες, μαζί με το bash script.

- Το πρόγραμμα δεν δέχεται πολλαπλά ορίσματα στις εντολές που κατατίθενται.

- Γίνεται δέσμευση χώρου για POOL_SIZE pools.

- Το job και το console κάνουν sleep(2) για να έχουν την ευκαιρία να αναδειχθούν 
  και οι υπόλοιπες λειτουργίες.

- Το όρισμα n, είναι υποχρεωτικό για το size του bash script, και επιπλέον, η λίστα για 
  τα directories που έχουν δημιουργηθεί από τις εργασίες, εμφανίζει ολόκληρο το path με αυτά.

- Για το bash script χρειάζεται να εκτελεστεί και το : sed -i 's/\r$//' jms_script.sh

- Στη λειτουργία "status", αν πρόκειται για ενεργές εργασίες, πρώτα εκτυπώνεται ο χρόνος 
  εκτέλεσης, και στη συνέχεια οι υπόλοιπες πληροφορίες

- Η υποβολή περαιτέρω εργασιών από το stdin, δεν είναι πλήρως αποτελεσματική.
